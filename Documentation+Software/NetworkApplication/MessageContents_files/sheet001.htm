Byte Meaning Recall that each byte ranges in value between 0 and 255 inclusive. There are 8 bits in a byte.
0 Message Length Total number of bytes in the message (LoRa P2P packet payload).
1 System ID A unique system identification number.
2 Source Node ID Relay nodes always have an ID of 0. They do not originate messages. They only rebroadcast. There can only be 255 non-relay nodes, 1 .. 255.
3 Destination Node ID
4 Source Message ID - High Byte High and low byte of the message ID. Message IDs are assigned in sequence. They reset when a value representable by a 16-bit unsigned integer is exceeded.
5 Source Message ID - Low Byte
6 Message Type (*) The type of message informs destination on how to interpret the contents.
7 Source Sensor ID There can be no more than 256 sensors connected to a given node. The Node ID is the high-byte and the Sensor ID is the low-byte in a 16-bit integer that identifies a specific sensor.
8 Remaining Rebroadcasts Relay nodes rebroadcast messages. Each rebroadcast decreases  this value. A message is rebroadcast at most 255 times. A value of zero indicates a message should not be rebroadcast.
(*) 000 = image's pixel-level data
003 = text message; for now, <sensor>:<value>:<sensor>:<value>…....
009 = reset message counter in tracking table for specified source node
           All nodes receiving this message reset their table if they use one
